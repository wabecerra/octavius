# Octavius Integration Fix — Task Specification

## Context
Octavius is a Next.js 14 life dashboard at `/home/wabo/workplace/ocbot/octavius`. It builds and tests pass (800/800). But multiple integration issues prevent it from working as a seamless OpenClaw extension.

## Issues to Fix

### 1. Auth chicken-and-egg problem
The Memory API (`/api/memory/*`) requires Bearer auth. The token is auto-generated in SQLite on first access. But the frontend (MemoryConfigSection, BiometricDataSection, SyncLayer, ChatPanel) makes API calls WITHOUT auth headers → gets 401 → silently fails.

**Fix:** Create a setup/bootstrap API route that:
- For same-origin requests from the dashboard: either skip auth OR use a session-based approach
- Simplest: Add a new env var `OCTAVIUS_API_SECRET` that, when set, is used. When NOT set, the Memory API should work without auth for same-origin requests (self-hosted app, single user).
- Update `src/app/api/memory/auth.ts` to allow unauthenticated requests when `OCTAVIUS_API_SECRET` is not set (dev/local mode)
- Create `.env.local` with a generated token for production use

### 2. Setup script (`scripts/setup.sh`)
Create an automated setup script that:
- Generates `OCTAVIUS_API_SECRET` token
- Creates `.env.local` from `.env.example` with the generated token
- Creates the SQLite `.data/` directory
- Detects if OpenClaw gateway is running (check port 18789)
- If gateway found, auto-configure gateway address/port
- Run `npm install` if node_modules missing
- Print clear instructions for next steps

### 3. Gateway connection
The dashboard's GatewayClient tries to connect to `localhost:18789` but:
- It doesn't send the correct auth token
- There's no auto-discovery flow

**Fix:** Update the gateway connection setup in the Settings tab to:
- Auto-detect the OpenClaw gateway on startup (check localhost:18789)
- Read the gateway token from OpenClaw config if accessible (try `~/.openclaw/openclaw.json`)
- Show connection status clearly (not just "unknown")
- The GatewayClient needs to try connecting on page load

### 4. Memory Config shows "coming soon" for SQLite
The MemoryConfigSection silently fails on 401 and shows defaults. Since SQLite actually works fine, the fix from item #1 will resolve this — once auth is fixed, the config will load from the actual database.

Add the missing config fields UI for:
- `reranking_enabled` (checkbox, auto-enabled when embeddings are on)
- `query_expansion_enabled` (checkbox, auto-enabled when embeddings are on)
- `smart_chunking_target_tokens` (number input)

### 5. SyncLayer auth
`src/lib/memory/sync-layer.ts` sends mutations to `/api/memory/items` without auth headers.

**Fix:** Add auth header support to SyncLayer. It should:
- Accept an optional auth token in constructor
- Include `Authorization: Bearer <token>` in all fetch calls
- The token should come from the auto-generated config or env var

### 6. Initial data seeding
On first run with empty data, the dashboard shows all zeros and "—" everywhere. Add a first-run experience:
- Show a welcome message on the Dashboard tab when no data exists
- Pre-populate the profile section with a prompt to fill it in
- Show helpful tooltips on empty states

### 7. Update README installation steps
The README says `pnpm install` but `package.json` uses `npm`. Standardize on npm.
Update Quick Start to include running the setup script:
```
git clone ...
cd octavius
npm run setup    # or: node scripts/setup.mjs
npm run dev
```
